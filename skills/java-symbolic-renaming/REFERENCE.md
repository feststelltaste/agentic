# Improve Naming – Reference

## OpenRewrite

Run with Maven:
```bash
./mvnw -U org.openrewrite.maven:rewrite-maven-plugin:run \
  -Drewrite.recipeArtifactCoordinates=org.openrewrite:rewrite-java:LATEST \
  -DactiveRecipes=RenameSymbol
```

Run with Gradle:
```bash
./gradlew rewriteRun
```

Place `rewrite.yml` at the project root.

### Rename a method

```yaml
---
type: specs.openrewrite.org/v1beta/recipe
name: RenameSymbol
recipeList:
  - org.openrewrite.java.ChangeMethodName:
      methodPattern: "com.example.MyClass oldMethodName(..)"
      newMethodName: newMethodName
      ignoreDefinition: false
```

`methodPattern` format: `fully.qualified.ClassName methodName(ArgType, ..)` — use `..` to match any args.
Set `ignoreDefinition: true` to only rename call sites (useful for interface rename before updating the impl).

### Rename a field

```yaml
---
type: specs.openrewrite.org/v1beta/recipe
name: RenameSymbol
recipeList:
  - org.openrewrite.java.RenameField:
      classPattern: "com.example.MyClass"
      fieldName: oldFieldName
      newFieldName: newFieldName
```

### Rename a class or interface

```yaml
---
type: specs.openrewrite.org/v1beta/recipe
name: RenameSymbol
recipeList:
  - org.openrewrite.java.ChangeType:
      oldFullyQualifiedTypeName: com.example.OldName
      newFullyQualifiedTypeName: com.example.NewName
```

This updates all usages, imports, and `instanceof` checks across the project.

### Rename a static constant

Use `RenameField` (same as field rename above). If the constant is part of a public API, consider a two-step: add the new constant, deprecate the old, then remove in a follow-up PR.

## ast-grep

Install: `npm i -g @ast-grep/cli` or `cargo install ast-grep`

Run: `ast-grep scan --rule rename.yml src/` (dry-run) or `ast-grep scan --rule rename.yml --rewrite src/`

### Rename a local variable within a method

```yaml
id: rename-local-var
language: Java
rule:
  pattern: oldName
  inside:
    kind: method_declaration
fix: newName
```

> **Warning**: `pattern: oldName` matches the bare identifier anywhere inside a method.
> Scope it further with an enclosing `inside` block matching the specific class and method if the name is common.

### Rename a parameter

```yaml
id: rename-parameter
language: Java
rule:
  any:
    - pattern: oldName          # usages in method body
    - pattern: $TYPE oldName    # declaration in signature
  inside:
    kind: method_declaration
    has:
      pattern: $TYPE oldName    # only methods that declare this param
fix: newName
```

### Rename a private field (within one class)

```yaml
id: rename-private-field
language: Java
rule:
  pattern: this.oldFieldName
fix: this.newFieldName
```

Run scoped to the single class file: `ast-grep scan --rule rename.yml src/com/example/MyClass.java`

## Comments and Javadoc

No tool renames identifiers inside comments. After any code rename, always run these searches and edit manually.

### Find all comment mentions

```bash
# Javadoc @param and @return mentioning the name
grep -rn "@param\s\+oldName\|@return.*oldName" src/

# Javadoc {@link} and {@code} references
grep -rn "{@link.*oldName\|{@code.*oldName" src/

# Javadoc @see references
grep -rn "@see.*oldName" src/

# Inline // comments
grep -rn "//.*oldName" src/

# Block /* */ comments
grep -rn "\*.*oldName" src/
```

### Javadoc tag cheatsheet

| Tag | Example before | Example after |
|---|---|---|
| `@param` | `@param oldName the count` | `@param newName the count` |
| `{@link}` | `{@link MyClass#oldMethod()}` | `{@link MyClass#newMethod()}` |
| `{@code}` | `{@code oldName}` | `{@code newName}` |
| `@see` | `@see MyClass#oldMethod` | `@see MyClass#newMethod` |

## Manual Checklist

When no automated tool is available, use this checklist:

1. **Find all declaration sites**
   ```bash
   grep -rn "\boldName\b" src/
   ```

2. **Find all usages** (including imports, annotations, string references)
   ```bash
   grep -rn "oldName" src/ test/
   ```

3. **Check for reflection** (string-based lookup)
   ```bash
   grep -rn '"oldName"' src/
   ```

4. **Check config / serialization** (JSON, XML, YAML field names that map to this symbol)
   ```bash
   grep -rn "oldName" src/main/resources/
   ```

5. **Check subclasses / interface implementations** (for method renames)
   ```bash
   grep -rn "oldName" src/ | grep -v "//.*oldName"
   ```

6. **Update comments and Javadoc** (see section above)

7. Make all edits, then run the full test suite before committing.

## Rename Script

When no tool is available, generate a `rename.sh` for the user to review and run.
Prefer scoped `find` targets over bare `src/` to avoid false positives.

```bash
#!/usr/bin/env bash
# Rename: oldName -> newName
# Generated by improve-naming skill — review before running!
set -euo pipefail

OLD="oldName"
NEW="newName"

# Scope to relevant paths (adjust as needed)
FILES=$(grep -rl --include="*.java" "\b${OLD}\b" src/)

if [ -z "$FILES" ]; then
  echo "No files found containing '${OLD}'"
  exit 0
fi

echo "Files to be modified:"
echo "$FILES"
echo ""
echo "Preview (first 20 matches):"
grep -rn --include="*.java" "\b${OLD}\b" src/ | head -20
echo ""
read -p "Proceed with rename? [y/N] " confirm
[[ "$confirm" == [yY] ]] || { echo "Aborted."; exit 1; }

# Rename in Java source files (whole-word match)
echo "$FILES" | xargs sed -i "s/\b${OLD}\b/${NEW}/g"

echo "Done. Run your tests before committing."
```

**Caveats:**
- `sed` word-boundary `\b` is not available on macOS's BSD sed — use `gsed` (install via `brew install gnu-sed`) or replace with `perl -pi -e`.
- Does not update comments (see [Comments and Javadoc](#comments-and-javadoc)).
- Does not handle file/class renames — rename files separately if needed.
